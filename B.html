B.1 Дополнительный синтаксис # Ⓣ Ⓔ ① Ⓐ

Предыдущие издания спецификации ECMAScript содержали дополнительный синтаксис и семантику для определения восьмеричных литералов и восьмеричных управляющих последовательностей. В настоящем издании спецификации ECMAScript они были удалены. Данное приложение, не являющееся нормативом, представляет унифицированный синтаксис и семантику для восьмеричных литералов и восьмеричных управляющих последовательностей в целях совместимости с некоторыми более ранними программами на ECMAScript.

B.1.1 Числовые литералы # Ⓣ Ⓔ ① Ⓐ

Синтаксис и семантика, изложенные в пункте 7.8.3, могут быть расширены, как указано ниже, за исключением того, что это расширение не допускается в коде в строгом режиме:

Синтаксис

NumericLiteral ::

DecimalLiteral
HexIntegerLiteral
OctalIntegerLiteral

OctalIntegerLiteral ::

0 OctalDigit
OctalIntegerLiteral OctalDigit

OctalDigit :: один из

0 1 2 3 4 5 6 7

Семантика

  • MV для NumericLiteral :: OctalIntegerLiteral равно MV для OctalIntegerLiteral.

  • MV для OctalDigit :: 0 равно 0.

  • MV для OctalDigit :: 1 равно 1.

  • MV для OctalDigit :: 2 равно 2.

  • MV для OctalDigit :: 3 равно 3.

  • MV для OctalDigit :: 4 равно 4.

  • MV для OctalDigit :: 5 равно 5.

  • MV для OctalDigit :: 6 равно 6.

  • MV для OctalDigit :: 7 равно 7.

  • MV для OctalIntegerLiteral :: 0 OctalDigit равно MV для OctalDigit.

  • MV для OctalIntegerLiteral :: OctalIntegerLiteral OctalDigit равно (MV для OctalIntegerLiteral, умноженное на 8) плюс MV для OctalDigit.

B.1.2 Строковые литералы # Ⓣ Ⓔ ① Ⓐ

Синтаксис и семантика, изложенные в пункте 7.8.4, могут быть расширены, как указано ниже, за исключением того, что это расширение не допускается в коде в строгом режиме:

Синтаксис

EscapeSequence ::

CharacterEscapeSequence
OctalEscapeSequence
HexEscapeSequence
UnicodeEscapeSequence

OctalEscapeSequence ::

OctalDigit [lookahead ∉ DecimalDigit]
ZeroToThree OctalDigit [lookahead ∉DecimalDigit]
FourToSeven OctalDigit
ZeroToThree OctalDigit OctalDigit

ZeroToThree :: один из

0 1 2 3

FourToSeven :: один из

4 5 6 7

Семантика

  • CV для EscapeSequence :: OctalEscapeSequence равно CV для OctalEscapeSequence.

  • CV для OctalEscapeSequence :: OctalDigit [lookahead ∉ DecimalDigit] является символом, кодовое значение которого равно MV для OctalDigit.

  • CV для OctalEscapeSequence :: ZeroToThree OctalDigit [lookahead ∉ DecimalDigit] является символом, кодовое значение которого равно (8 умноженное на MV для ZeroToThree) плюс MV для OctalDigit.

  • CV для OctalEscapeSequence :: FourToSeven OctalDigit является символом, кодовое значение которого равно (8 умноженное на MV для FourToSeven) плюс MV для OctalDigit.

  • CV для OctalEscapeSequence :: ZeroToThree OctalDigit OctalDigit является символом, кодовое значение которого равно (64 (то есть, 82) умноженное на MV для ZeroToThree) плюс (8 умноженное на MV для первого OctalDigit) плюс MV для второго OctalDigit.

  • MV для ZeroToThree :: 0 равно 0.

  • MV для ZeroToThree :: 1 равно 1.

  • MV для ZeroToThree :: 2 равно 2.

  • MV для ZeroToThree :: 3 равно 3.

  • MV для FourToSeven :: 4 равно 4.

  • MV для FourToSeven :: 5 равно 5.

  • MV для FourToSeven :: 6 равно 6.

  • MV для FourToSeven :: 7 равно 7.

B.2 Дополнительные свойства # Ⓣ Ⓔ ① Ⓐ

Некоторые реализации ECMAScript включают дополнительные свойства для некоторых из стандартных родных объектов. Данное приложение, не являющееся нормативом, предлагает унифицированную семантику для таких свойств, при этом данные свойства или их семантика не становятся частью данного стандарта.

B.2.1 escape (string) # Ⓣ Ⓔ ① Ⓐ

Функция escape является свойством глобального объекта. Она вычисляет новую версию строкового значения, в котором определённые значения заменяются шестнадцатеричной управляющей последовательностью.

Для заменяемых символов, кодовое значение которых равно 0xFF или менее, используется двузначная управляющая последовательность в формате % xx. Для заменяемых символов, кодовое значение которых больше 0xFF, используется четырёхзначная управляющая последовательность в формате %u xxxx.

При вызове функции escape с одним аргументом string выполняются следующие шаги:

  1. Вызвать ToString(string).

  2. Вычислить количество символов в Result(1).

  3. Пусть R будет пустой строкой.

  4. Пусть k будет 0.

  5. Если k равно Result(2), вернуть R.

  6. Получить символ (представленый в виде 16-битового беззнакового целого) в позиции k в Result(1).

  7. Если Result(6) равен одному из 69 непробельных символов
    “ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789@*_+-./”,
    перейти к шагу 13.

  8. Если Result(6) меньше чем 256, перейти к шагу 11.

  9. Пусть S будет строкой, содержащей шесть символов “%u wxyz”, где wxyz представляют собой четыре шестнадцатеричных числа, которыми закодировано значение Result(6).

  10. Перейти к шагу 14.

  11. Пусть S будет строкой, содержащей три символа “% xy”, где xy представляют собой два шестнадцатеричных числа, которыми закодировано значение Result(6).

  12. Перейти к шагу 14.

  13. Пусть S будет строкой, содержащей один символ Result(6).

  14. Пусть R будет новым строковым значением, полученным в результате конкатенации S с предыдущим значением R.

  15. Увеличить k на 1.

  16. Перейти к шагу 5.

ПРИМЕЧАНИЕ Данный конкретный метод кодирования частично основан на кодировании, описанном в RFC 1738. Однако весь метод кодирования, заданный данным стандартом, описывается выше безотносительно содержания RFC 1738. Данное кодирование не отражает изменений, внесенных в версию RFC 1738 более поздней версией RFC 3986.

B.2.2 unescape (string) # Ⓣ Ⓔ ① Ⓐ

Функция unescape является свойством глобального объекта. Она вычисляет новую версию строкового значения, в котором каждая управляющая последовательность, которая может быть введена посредством функции escape, заменяется символом, который она представляет.

При вызове функции unescape с одним аргументом string выполняются следующие шаги:

  1. Вызвать ToString(string).

  2. Вычислить количество символов в Result(1).

  3. Пусть R будет пустой строкой.

  4. Пусть k будет 0.

  5. Если k равно Result(2), вернуть R.

  6. Пусть c будет символом в позиции k в Result(1).

  7. Если c не равно %, перейти к шагу 18.

  8. Если k больше чем Result(2)−6, перейти к шагу 14.

  9. Если символ в позиции k+1 в Result(1) не равен u, перейти к шагу 14.

  10. Если четыре символа в позициях k+2, k+3, k+4 и k+5 в Result(1) не являются шестнадцатеричными числами, перейти к шагу 14.

  11. Пусть c будет символом, кодовое значение которого является целым числом, представленным четырьмя шестнадцатеричными числами в позициях k+2,k+3, k+4 и k+5 в Result(1).

  12. Увеличить k на 5.

  13. Перейти к шагу 18.

  14. Если k больше чем Result(2)−3, перейти к шагу 18.

  15. Если два символа в позициях k+1 и k+2 в Result(1) оба не являются шестнадцатеричными числами, перейти к шагу 18.

  16. Пусть c будет символом, кодовое значение которого является целым числом, представленным двумя нулями плюс двумя шестнадцатеричными числами в позициях k+1 и k+2 в Result(1).

  17. Увеличить k на 2.

  18. Пусть R будет новым строковым значением, полученным в результате конкатенации c с предыдущим значением R.

  19. Увеличить k на 1.

  20. Перейти к шагу 5.

B.2.3 String.prototype.substr (start, length) # Ⓣ Ⓔ ① Ⓐ

Метод substr принимает два аргумента start и length и возвращает подстроку, полученную в результате преобразования в строку этого объекта, начиная с позиции start и до символов length (или до конца строки, если length равно undefined). Если start имеет отрицательное значение, вместо него используется sourceLength+start, где sourceLength является длиной строки. Результатом является строковое значение, а не объект String. Выполняются следующие шаги:

  1. Вызвать ToString, передавая значение this в качестве аргумента.

  2. Вызвать ToInteger(start).

  3. Если length равно undefined, использовать +∞; иначе вызвать ToInteger(length).

  4. Вычислить количество символов в Result(1).

  5. Если Result(2) равно положительному числу или нулю, использовать Result(2); иначе использовать max(Result(4)+Result(2),0).

  6. Вычислить min(max(Result(3),0), Result(4)–Result(5)).

  7. Если Result(6) ≤ 0, вернуть пустую строку "".

  8. Вернуть строку, содержащую последовательные символы в количестве Result(6) из Result(1), начиная с символа в позиции Result(5).

Свойство length для метода substr равно 2.

ПРИМЕЧАНИЕ Функция substr специально является родовой функцией. Она не требует, чтобы её значение this было объектом типа String. Поэтому она может передаваться другим типам объектов для использования в качестве метода.

B.2.4 Date.prototype.getYear ( ) # Ⓣ Ⓔ ① Ⓐ

ПРИМЕЧАНИЕ Практически для всех целей предпочтителен метод getFullYear, поскольку он позволяет избежать "проблемы 2000 года".

При вызове метода getYear без аргументов выполняются следующие шаги:

  1. Пусть t будет числовым значением this.

  2. Если t равно NaN, вернуть NaN.

  3. Вернуть YearFromTime(LocalTime(t)) − 1900.

B.2.5 Date.prototype.setYear (year) # Ⓣ Ⓔ ① Ⓐ

ПРИМЕЧАНИЕ Практически для всех целей предпочтителен метод setFullYear, поскольку он позволяет избежать "проблемы 2000 года".

При вызове метода setYear с одним аргументом year выполняются следующие шаги:

  1. Пусть t будет результатом LocalTime(значение времени this ); но если значение времени this равно NaN, пусть t будет +0.

  2. Вызвать ToNumber(year).

  3. Если Result(2) равно NaN, установить внутреннее свойство [[PrimitiveValue]] значения this в NaN и вернуть NaN.

  4. Если Result(2) не равно NaN и 0 ≤ ToInteger(Result(2)) ≤ 99, то Result(4) равно ToInteger(Result(2)) + 1900. Иначе Result(4) равно Result(2).

  5. Вычислить MakeDay(Result(4), MonthFromTime(t), DateFromTime(t)).

  6. Вычислить UTC(MakeDate(Result(5), TimeWithinDay(t))).

  7. Установить внутреннее свойство [[PrimitiveValue]] значения this в TimeClip(Result(6)).

  8. Вернуть значение внутреннего свойства [[PrimitiveValue]] значения this.

B.2.6 Date.prototype.toGMTString ( ) # Ⓣ Ⓔ ① Ⓐ

ПРИМЕЧАНИЕ Предпочтительным является свойство toUTCString. Свойство toGMTString предоставляется в основном в целях совместимости с более ранним кодом. В новом коде на ECMAScript рекомендуется использовать свойство toUTCString.

Объект типа Function, представляющий собой начальное значение для Date.prototype.toGMTString является тем же объектом Function, который представляет собой начальное значение для Date.prototype.toUTCString.
